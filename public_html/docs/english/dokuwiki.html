Geeklog DokuWiki Plugin Documentation

The following information explains the configuration options, autotags and templates of the Geeklog DokuWiki plugin. For install or upgrade instructions please visit the DokuWiki GitHub page.

For all usage questions and documentation on how to utilize DokuWiki, please refer to the DokuWiki Web Site at: https://www.dokuwiki.org/wiki:dokuwiki

Configuration

Most DokuWiki configuration options is accessible via DokuWiki's own Administration page (Configuration Settings) found under the Geeklog Command and Control panel.

The configuration relating directly to the DokuWiki plugin which affects Geeklog can be changed from the Geeklog Configuration admin panel:

Template Files

DokuWiki's layout can be customized through templates (aka. skins). The DokuWiki Integration Plugin must use a customized DokuWiki template to properly integrate the look and feel of the Geeklog site. Included with the plugin are two customized skins; glfusion and dokuwiki. glfusion is the original skin used by DokuWiki until late 2012. The dokuwiki skin is a customized version of the standard DokuWiki theme that works with Geeklog.

To update the layout of your DokuWiki template, you can change some settings via DokuWiki's own Administration. If more advanced changes are required you can modify the template files directly. The tempalte files are located here:

/path/to/geeklog/public_html/dokuwiki/lib/tpl

Autotags

The DokuWiki Plugin provides one autotag, [wiki:]. You can use this autotag to display a link to a wiki page using the wiki page title as the link text. An alternate title may be specified but is not required.

For example, the following autotags will translate to:

  • [wiki:start] will be translated to: <a href="http://www.yoursite.com/dokuwiki/doku.php?id=start">start</a>
  • [wiki:start click to visit wiki page] will be translated to: <a href="http://www.yoursite.com/dokuwiki/doku.php?id=start">click to visit wiki page</a>
